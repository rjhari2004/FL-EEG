# FL-EEG
Federated Learning Based Seizure Detection Using EEG 
# 🧠 Federated Seizure Detection: Hybrid CA-Matrix & pFL

This repository contains the complete implementation for a decentralized, privacy-preserving EEG seizure detection system. It utilizes **FedProx** for global model aggregation and **Personalized Federated Learning (pFL)** for patient-specific optimization using the **CHB-MIT Scalp EEG Database**.

---

## 🚀 Key Research Contributions

* **Hybrid CA-Matrix Projection:** Implements hardware-efficient dimensionality reduction using **Hybrid Cellular Automata** to project spectral features.
* **Mish-Attention Architecture:** A sophisticated classifier combining **Mish activation**, **Self-Attention**, and **Residual Blocks** optimized for EEG time-series.
* **Non-Overlapping Windowing:** Uses a strict **6-second non-overlapping window** strategy ($1536$ samples at $256$ Hz) to ensure zero data leakage and high clinical validity.
* **Surgical Personalization:** Employs a **5-epoch "Calibration" phase** with a stabilized learning rate ($1 \times 10^{-5}$) to provide a significant performance boost without catastrophic forgetting.

---

## 🛠️ System Architecture & Workflow

### 1. Data Pipeline

The system processes 23-channel EEG data. It balances classes (Normal vs. Pre-ictal) using a `WeightedRandomSampler` to ensure that accuracy metrics (often 90%+) are scientifically robust and not biased by class imbalance.

### 2. Federated Training

* **Global Rounds:** 15 rounds of **FedProx** to learn generalized features across the diverse patient population while handling system heterogeneity.
* **Personalization:** Post-FL local fine-tuning where the model adapts to the specific neural signature of the edge device.

### 3. Docker-Automated Environment

The project is containerized for seamless deployment on Windows/WSL2 or Linux.

* **Cleanup Service:** Automatically wipes the `./runs` directory at the start of every session to prevent stale data from previous experiments.
* **Polling Mechanism:** Uses the `--server.fileWatcherType poll` flag to ensure the Streamlit dashboard reloads instantly across the Docker "volume bridge".

---

## 📈 Dashboard & Real-Time Monitoring

The project includes a **Streamlit Dashboard** for real-time monitoring of the federated process.

* **URL:** `http://localhost:8501`
* **Heartbeat Mechanism:** A custom background trigger forces the UI to refresh every few seconds, catching new patient reports as they are generated by the edge clients.
* **Metric Comparison:** Displays side-by-side classification reports (Precision, Recall, F1) for the **Global Model** vs. the **Personalized Model**.

## 🚦 Getting Started

### Prerequisites

* Docker & Docker Compose
* Python 3.10+ (for local development)

### Deployment

1. **Clone the Repository**
2. **Start the Simulation:**
```bash
docker-compose up --build

```


3. **Monitor Progress:** Open your browser to `localhost:8501` to view the live personalization boost as each patient completes their 5-epoch calibration.
